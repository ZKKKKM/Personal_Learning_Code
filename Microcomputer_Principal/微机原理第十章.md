
8253/8254工作方式

| **模式** | 启动方式                          | 停止计数     | 自动计数 | 刷新初始值    | 输出波形                  |
| ------ | ----------------------------- | -------- | ---- | -------- | --------------------- |
| 方式0    | **软件启动（software）**            | GATE = 0 | 不是   | 立刻刷新     | 边沿上的可变延迟时间上升沿         |
| 方式1    | **硬件启动（hardware）**            | /        | 不是   | 在下一个周期刷新 | 宽度为 N × TCLK 的单个负脉冲。  |
| 方式2    | **软件硬件启动（software hardware）** | GATE = 0 | 是    | 在下一个周期刷新 | 周期为 N × TCLK 的连续负脉冲宽度 |
| 方式3    | **软件硬件启动（software hardware）** | GATE = 0 | 是    | 在下一个周期刷新 | 周期为 N × TCLK 的连续方波。   |
| 方式4    | **软件启动（software）**            | GATE = 0 | 不是   | 在下一个周期刷新 | 宽度为 TCLK 的单个负脉冲。      |
| 方式5    | **硬件启动（hardware)**            | /        | 不是   | 在下一个周期刷新 | 宽度为 TCLK 的单个负脉冲。      |

8255的工作方式

| 模式  | 模式名称                                       | 使用端口         | 主要功能                                                | C口的角色                                                |
| --- | ------------------------------------------ | ------------ | --------------------------------------------------- | ---------------------------------------------------- |
| 方式0 | 基本输入/输出模式 （Basic input / output mode）      | A、B、C 端口均可   | 最基础的 I/O，不需要直接的联络信号（握手信号）。端口可以被设计为独立的输入或输出。         | 作为普通的 8 位（或两个 4 位）数据端口使用，用于基本的输入输出。                  |
| 方式1 | 选通输入/输出模式 （Strobe input / output）          | 仅 A 端口和 B 端口 | 接口与外设之间需要通过联络信号（Contact Signal）进行协调。实现有“握手”协议的数据传输。 | C 口的部分引脚被定义为 A 口和 B 口的联络信号线；剩余引脚仍可工作在方式 0。           |
| 方式2 | 双向传输模式 （Bidirectional input / output mode） | 仅 A 端口       | 允许 A 端口在单一模式下既能输入也能输出（双向总线）。                        | C 口有 5 根线被用作 A 口的双向联络信号线；剩余 3 根线可作为 B 口的联络线或工作在方式 0。 |
